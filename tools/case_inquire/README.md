# 案例查询智能体

## 一、应用描述

案例查询智能体结合qwen，依托内置知识库与文档标签检索能力，实现对已有案例的快速精准查询。该智能体无需调用外部工具，仅通过匹配知识库中案例的标签特征与用户查询需求，即可提取案例关键信息并反馈查询结果，适用于企业过往项目案例、法律诉讼案例、业务处理案例等各类存量案例的检索与查阅场景。

## 二、应用功能

案例查询智能体应用主要包含以下几个大功能：
- 文档标签提取：通过 AI 提取出用户问题中所包含的标签，从而使标签能够独立出来进行调用以及进行文档标签检索。
- 文档标签检索匹配：基于知识库中已标注的案例标签（如产品、行业、资本成分、部署方式等），智能体自动检索并匹配与用户需求相符的案例，实现快速精准定位。
- 案例查询：基于知识库中的查询到的文档进行结果总结，给出相关案例信息。

## 三、应用构建要素

案例查询智能体应用构建时涉及的核心要素内容：
- 大模型：qwen-max 、qwen3-next-80b；
- 知识库：案例知识库；
- 提示词示例：案例查询总结提示词；

```提示词
	# 已知信息：
	{{知识库检索.data}}
	
	# 用户问题：
	{{开始.question}}
	
	# 要求
	- 1. 根据已知信息回答用户问题。如果已知信息中没有答案，则以委婉且诚实的回答表示不知道即可。
	- 2. 案例总结必须以*优美的形式*输出
	- 3. 必须严格按照我给你的示例进行输出。
	- 4. 已知信息中可能存在不相关信息，必须进行筛选，只利用符合要求的信息。例如当我们查询物流行业有哪些案例时，可能存在多个查询结果，同时，行业的表述可能不完全一致，符合问题描述的行业的信息都应该予以采纳。例如，物流行业在实际的知识库检索中可能的表述有：物流科技、快递物流、冷链物流等等，这些应都属于物流行业，其他行业以此类推，但对其中不符合要求的信息不予采纳。
	- 5. 如果用户问题是具体问题，在示例中的查询结果下则需要回答用户的具体问题，回答要求清晰直接，例如当问某行业有哪些客户时应当回答出具体的客户名称，一次类推。
	- 6. 在回答用户问题之后，如果已知信息中包括原文链接，则在回答的开始按照示例格式附上原文链接。原文链接必须与用户问题有关且来源于已知信息中找到的内容；如果没有，则不必输出原文链接。
	- 7. 基于已知信息对如下字段进行总结，未提及的信息则无需总结。有的信息总结为一段话。记住，必须是一段话！需要总结的字段如下，并将其放入示例中且紧跟着字段不应换行（🔲也必须输出）：
	'''
	 客户名称（工商注册）:
	■ 客户城市:
	■ 客户行业 :
	■ 资本属性 :
	■ 是否上市 :
	■ 主营业务:
	■ 签约产品:
	■ 购买类型：替换/新购
	■ 竞品情况:
	■ 客户简介:
	■ 客户痛点:
	■ 关注功能点:
	■ 关键场景:
	■ 部署方式:
	■ 典型收益:
	'''
	
	# 示例
	# 查询结果（如果问题是具体问题 ps：括号内不要输出）
	（这里需要直接回答用户的具体问题）
	
	## 案例1: 案例名称（如果有原文链接的情况 ps：完全按照引用知识库的案例名称）
	
	### 原文链接
	https://www.example.com
	
	### 案例总结
	你总结的相关信息。
	
	## 案例2：案例名称（没有原文链接的情况 ps：括号内不要输出）
	
	### 案例总结
	你总结的相关信息。
	
	以此类推
	
	
	# 限制
	输出里包含“未提及”字段的内容不要输出
```
