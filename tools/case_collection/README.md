# 案例收集智能体

## 一、应用描述

案案例收集智能体聚焦多来源案例的标准化收集与知识库入库需求，支持 URL 链接、文档（Word、PDF 等）、视频三类核心案例来源。该智能体通过针对性解析技术提取不同来源的案例核心信息，联动 MCP 及工具补全缺失数据，同时基于案例内容自动生成多维度精准标签，最终输出标准化案例信息并录入知识库，为后续案例查询、复用提供全面且高质量的数据支撑。

## 二、应用功能

案例收集智能体应用主要包含以下几个大功能：
- 多来源案例上传与解析：支持 URL 链接、文档、视频三类案例来源的接入与解析，针对性提取原始信息，形成初步案例信息草稿
- 信息补充与完善：调取Cordys CRM的MCP服务以及工具商店的企业工商信息查询工具完善客户信息。
- 智能标签生成：基于解析后的案例原始信息、补充完善的数据，自动匹配预设标签体系，为案例打上多维度标签，确保标签精准贴合案例核心特征
- 标准化案例生成与知识库入库：将整合后的完整信息按预设模板整理为标准化案例文档，包含全部核心字段（客户名称、客户城市、行业、资本属性、签约产品、客户痛点、典型收益等）及对应标签，确保格式统一、信息完整。经格式校验无误后，自动录入案例知识库，支持后续按标签检索、字段筛选等应用场景，同时记录案例原始来源，便于溯源。

## 三、应用构建要素

案例收集智能体应用构建时涉及的核心要素内容：
- 大模型：qwen-max 、qwen3-next-80b；
- MCP与工具：Cordys CRM、企业工商信息查询;
- 知识库：提前创建好的知识库ID；
- 子应用：案例收集智能体 子应用、CordysCRM查询案例

## 五、注意事项

- 嵌套关系是`案例收集智能体`调用`案例收集智能体子应用`，`案例收集智能体子应用`调用`CordysCRM查询案例`
- 应当事先准备好企业工商信息查询API
- 由于该智能体是直接指定知识库导入，所以应当准备好知识库ID
- 知识库中的标签必须提前创建好，该智能体无法录入知识库中没有的标签。标签录入的逻辑是将通过把`即将录入的标签值`与`知识库中已有的标签值`做匹配提取出`知识库中标签值ID`，通过`标签值ID`把标签录入知识库
