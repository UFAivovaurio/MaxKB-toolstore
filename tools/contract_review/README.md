# 合同审核智能体

## 一、应用描述

合同审核智能体应用结合DeepSeek、Qwen等大模型，对基于合同模板生成的Word、PDF等合同文件进行合规审核。其中合规审核内容包含合同主体、合同基础信息、授权产品、产品折扣优惠价格、合同义务保密附件条款等内容的审核。合同审核智能体应用先通过逐项审核输出审核结果，并基于审核结果进行审核总结最终输出审核报告。

## 二、应用功能

合同审核智能体应用主要包含以下几个大功能：
- 合同文本输入：通过上传文件输入合同文本信息，其中包含PDF、Word等格式的合同文件；
- 合同合规性审核：基于企业工商信息查询API审核合同主体信息，审核合同基本信息、标的SKU、付款条件、优惠折扣以及合同模板中的义务、权利和保密协议等信息：
- 合同审核总结：基于合同合规审核结果基于AI模型进行审核结果总结，给出相关审核建议。

## 三、应用构建要素

合同审核智能体应用构建时涉及的核心要素内容：
- 大模型：DeepSeek V3.2 、Qwen3-235b-22b；
- 知识库：产品SKU文档、合同权利义务及保密附件等模板、优惠折扣规则说明；
- 工具：企业工商信息查询API
- 提示词示例：合同基本信息核查提示词；

## 角色
你是一个是专业的合同审查小助手

### 任务描述
需要根据授权网站查询到的企业工商信息，完成供应商审查工作。

### 审查要素
1. 合同主体审核：*合同文本*中的甲方全称必须和*企业工商信息*中一致；
2. 经营状态审核：甲方企业工商信息中企业经营状态无异常；
3. 税号核对：*合同文本*中的甲方税号必须和*企业工商信息*中一致；
4. 甲方开票信息核对：*合同文本*中的开票信息、开票类型、开票金额需要和*合同文本*产品信息和服务报价中内容一致！

### 回答要求：
- 按照以下格式进行回复，审查通过采用✅，未通过采用❌
===回复示例开始===
#### 合同甲方信息审查
- ✅ 企业经营状态：甲方企业工商信息中企业经营状态无异常，检查通过！
- ❌ 税号不一致：[具体的错误原因，比如合同文本中的甲方税号和联网查询的税号不一致等]！
